HTTP Basics

Learn how web browsers and servers communicate using the Hypertext Transfer Protocol.

What is HTTP?

HTTP stands for Hypertext Transfer Protocol. It is the standard communication protocol used by web browsers and web servers to exchange information over the internet.

Every time you visit a website, your browser sends an HTTP request to a server. The server processes that request and returns an HTTP response containing the requested webpage, images, videos, or other files.

How HTTP Works

  1. You enter a website address (URL).
  2. Your browser sends an HTTP request.
  3. The server receives and processes the request.
  4. The server sends an HTTP response.
  5. Your browser displays the webpage.

This entire process usually takes only a fraction of a second.

Client-Server Model

HTTP uses a client-server architecture. The client is usually a web browser, while the server stores website files and responds to requests.

Client Server
Sends requests Processes requests
Displays webpages Stores website files
Receives responses Sends responses

Common HTTP Methods

Method Purpose
GET Retrieves information from a server.
POST Sends data to a server.
PUT Updates existing information.
DELETE Removes information from a server.

Common HTTP Status Codes

Status Code Meaning
200 OK - The request was successful.
301 Moved Permanently.
403 Forbidden.
404 Page Not Found.
500 Internal Server Error.

Why HTTP Matters

  • Loads websites and web applications.
  • Transfers images, videos, and files.
  • Supports APIs used by mobile apps.
  • Allows communication between browsers and servers.
  • Makes online shopping, banking, and streaming possible.

HTTP is Stateless

HTTP does not automatically remember previous requests. Every request is treated as a completely new interaction between the browser and the server.

Websites use cookies and sessions to remember users after they log in or add items to an online shopping cart.
